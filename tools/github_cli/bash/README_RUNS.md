# GitHub CLI Workflow Tools

GitHub Actionsのワークフロー実行（run）を管理するためのツール群。

## ツール一覧

### `gpci.sh` - GitHub PR CI
現在のPRのCI状況を確認。

**使い方:**
```bash
gpci.sh [options]
```

**機能:**
- 現在のPRのCIチェック状況を表示
- `-e`オプションで失敗したCIのログのみを表示

**オプション:**
- `e`: エラーログのみを表示（失敗したワークフローの詳細ログ）
- `s`: リポジトリをfzfで選択
- `数字`: 取得する実行数の上限を指定

**例:**
```bash
# 現在のPRのCI状況を表示
gpci.sh

# 失敗したCIのエラーログのみを表示
gpci.sh e

# リポジトリを選択してCI状況を確認
gpci.sh s
```

---

### `grf.sh` - GitHub Run Failed
失敗したワークフロー実行を検索して詳細ログを表示。

**使い方:**
```bash
grf.sh [options]
```

**機能:**
- 失敗したワークフロー実行のみを一覧表示
- fzfでインタラクティブに選択
- プレビューで失敗ログを確認
- 選択後、失敗したステップの詳細ログを表示

**オプション:**
- `s`: リポジトリをfzfで選択
- `数字`: 取得する実行数の上限を指定（デフォルト: 10000）

**例:**
```bash
# 現在のリポジトリの失敗したrunを表示
grf.sh

# リポジトリを選択して失敗したrunを表示
grf.sh s

# 最新100件の失敗したrunから選択
grf.sh 100
```

---

### `grl.sh` - GitHub Run Logs
ワークフロー実行のログを表示。

**使い方:**
```bash
grl.sh [options]
```

**機能:**
- すべてのワークフロー実行を一覧表示
- fzfでインタラクティブに選択
- プレビューでログを確認
- 選択後、完全なログを表示
- `-e`オプションで失敗したrunのエラーログのみを表示

**オプション:**
- `e`: エラーログのみを表示（失敗したrunのみをフィルタ）
- `s`: リポジトリをfzfで選択
- `数字`: 取得する実行数の上限を指定（デフォルト: 10000）

**例:**
```bash
# 現在のリポジトリのrunログを表示
grl.sh

# 失敗したrunのエラーログのみを表示
grl.sh e

# 最新50件から選択
grl.sh 50
```

---

### `grs.sh` - GitHub Run Status
ワークフロー実行のステータス一覧を表示。

**使い方:**
```bash
grs.sh [options]
```

**機能:**
- ワークフロー実行の一覧を表形式で表示
- ステータス、結論、実行時間などを確認
- `-e`オプションで失敗したrunのみを表示

**オプション:**
- `e`: 失敗したrunのみを表示
- `s`: リポジトリをfzfで選択
- `数字`: 取得する実行数の上限を指定（デフォルト: 10000）

**例:**
```bash
# 現在のリポジトリのrun一覧を表示
grs.sh

# 失敗したrunのみを表示
grs.sh e

# リポジトリを選択して一覧を表示
grs.sh s

# 最新20件のrunを表示
grs.sh 20
```

## 共通仕様

### `-e` オプション（エラーのみ表示）
すべてのツールで`e`オプションをサポート：
- **`gpci.sh e`**: 現在のPRで失敗したCIのエラーログのみを表示
- **`grl.sh e`**: 失敗したワークフロー実行のみをfzfで選択し、エラーログを表示
- **`grs.sh e`**: 失敗したワークフロー実行のみを一覧表示

### リポジトリの選択
- デフォルトでは現在のgitリポジトリを使用
- `s`オプションで明示的にリポジトリを選択可能
- リポジトリが検出できない場合は自動的に選択画面が表示される

### fzfプレビュー
- `grf.sh`と`grl.sh`ではfzfのプレビュー機能でログを事前確認可能
- プレビューウィンドウは右側60%に表示

### 設定ファイル
`config.sh`で以下の設定を変更可能:
- `Limit1`: リポジトリ一覧の取得上限
- `Limit2`: ワークフロー実行の取得上限
- `error_only_flg`: エラーのみ表示フラグ（デフォルト: false）

## 依存関係
- `gh` (GitHub CLI)
- `fzf` (コマンドラインファジーファインダー)
- `git`

## ディレクトリ構造
```
github_cli/bash/
├── config.sh      # 共通設定
├── option.sh      # オプション解析（e オプション追加）
├── gpci.sh        # PR CI状況確認（e オプション対応）
├── grf.sh         # 失敗したrunを検索
├── grl.sh         # runのログを表示（e オプション対応）
└── grs.sh         # runのステータス一覧（e オプション対応）
```

## 使用例

### ワークフロー: PRのCI失敗を調査

```bash
# 1. 現在のPRのCI状況を確認
gpci.sh

# 2. 失敗しているCIのエラーログを詳細表示
gpci.sh e

# 3. 失敗したワークフロー実行を検索
grf.sh

# 4. 過去の失敗したrunの一覧を確認
grs.sh e
```

### ワークフロー: 特定のrunのログを確認

```bash
# 1. run一覧を表示
grs.sh

# 2. ログを詳細表示（fzfで選択）
grl.sh

# 3. 失敗したrunのエラーログのみ確認
grl.sh e
```
